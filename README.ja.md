## A. アイデンティティと権限

*対応する脅威: TH-05 エージェントのアイデンティティ/権限の濫用、TH-06 委任の濫用。マニュアル: 第4章。*

- [ ] **1.** すべてのエージェントは**専用のエージェント/ワークロードアイデンティティ**で動作しているか — 人間のユーザーの常設クレデンシャルを直接保持していないか？ `REQ-001 · TH-05 → 第4章`
- [ ] **2.** すべての委任された権限は、**目的・期間・対象リソース・許可される操作・承認者・失効条件**を伴っているか？ `REQ-002 · TH-05 → 第4章`
- [ ] **3.** 高リスク特権（本番変更、権限変更、外部送信、支払い、削除、シークレットアクセス）は、セッション全体を通じて常在させるのではなく、**使用直前に狭い範囲で**付与されているか？ `REQ-003 · TH-05, TH-08 → 第4章`
- [ ] **4.** エージェントが子エージェントや外部エージェントに委任するとき、子は**親より厳密に狭いスコープの新しいクレデンシャル**を受け取っているか（親のクレデンシャルの素通しは不可）。そして委任チェーン全体が単一の相関IDで監査可能か？ `REQ-051 · TH-06 → 第4章`

## B. ツールとアクションの安全性

*対応する脅威: TH-02 ツールの濫用/権限昇格、TH-07 サプライチェーン/MCP の侵害、TH-08 データ持ち出し。マニュアル: 第5章。*

- [ ] **5.** エージェントが利用できるすべてのツールは**明示的なアローリスト**に載っており、ネットワーク接続・ファイル書き込み・コード実行・資金移動が可能なツールはリスク分類されているか？ `REQ-010 · TH-02 → 第5章`
- [ ] **6.** 各ツールの権限は**その機能に必要な最小限**に絞られ、入力は実行前に**スキーマで検証**されているか？ `REQ-011 · TH-02 → 第5章`
- [ ] **7.** コード実行・ブラウジング・ファイル操作は、**本番環境・クレデンシャル・ネットワークからデフォルトで隔離されたサンドボックス**内で実行され、外向き通信は宣言済みの宛先に限定されているか？ `REQ-012 · TH-02, TH-08 → 第5章`
- [ ] **8.** 高インパクトなアクションの実行前に、人間は曖昧な要約ではなく**具体的な効果** — diff またはペイロード、宛先、行使される特権、ロールバック条件 — を提示されているか？ `REQ-015 · TH-02 → 第5章`
- [ ] **9.** MCP サーバーは**審査済みアローリスト**からのみ受け入れられ、ツール定義は**バージョン固定・ハッシュ化**されており、承認済み定義の無断変更（ラグプル）は使用前に検知・再レビューされるか？ `REQ-050 · TH-07 → 第5章`

## C. 信頼できないコンテンツ・RAG・メモリ

*対応する脅威: TH-01 プロンプトインジェクション、TH-03 RAG ポイズニング、TH-04 メモリ汚染。マニュアル: 第6章。*

- [ ] **10.** 検索・ツール・メモリからモデルに入るコンテンツは**ソースと信頼レベルでラベル付け**されているか — そして untrusted とラベル付けされたコンテンツは、システム指示・開発者ポリシー・ツールスキーマを構造的に上書きできないようになっているか？ `REQ-020 · TH-01, TH-03 → 第6章`
- [ ] **11.** 検索・記憶されたコンテンツは **TTL（有効期限）と鮮度シグナル**を伴い、古いコンテンツは無期限に信頼されるのではなく再検証または失効されるか？ `REQ-021 · TH-03 → 第6章`
- [ ] **12.** 長期メモリ・プロファイル・共有コンテキストへの書き込みは、**来歴タグ付き・ポリシー管理下・可逆**か？ `REQ-022 · TH-04 → 第6章`

## D. モニタリング・評価・インシデント対応

*対応する脅威: TH-08 データ持ち出し、TH-09 監査/評価の回避、TH-10 モデル/サービスの濫用。マニュアル: 第7章。*

- [ ] **13.** ログから**インシデントをエンドツーエンドで再構築**できるか — 入力、モデル応答、ツール呼び出しと結果、行為主体のアイデンティティ、承認、データアクセス、外部送信、最終出力 — シークレットは秘匿した上で？ `REQ-030 · TH-02, TH-08, TH-09 → 第7章`
- [ ] **14.** **エージェント固有のシグナル** — 異常なツール使用、通常と異なる権限要求、未申告の外部送信、インジェクションの兆候、拡大していく委任 — を監視し、アラートを上げているか？ `REQ-031 · TH-09 → 第7章`
- [ ] **15.** 脅威ファミリーをカバーする評価は、**継続的かつ再現可能に** — ローンチ前、変更時、定期スケジュールで — 実行され、失敗ケースは保持されて再実行されているか？ `REQ-033 · TH-09 → 第7章`
- [ ] **16.** ロールバックとリカバリは、**設計されているだけでなく実際に演習されている**か — 状態復元、メモリ汚染の巻き戻し、ツールバージョンのロールバック、クレデンシャル失効の検証？ `REQ-034 · TH-04 → 第7章`
- [ ] **17.** インシデント対応手順は、**エージェントの停止、ケイパビリティ粒度でのクレデンシャル失効、メモリとログの隔離・保全、委任をまたいだ影響範囲の特定**ができるか？ `REQ-035 · TH-10 → 第7章`

## E. ランタイムポスチャとガバナンス

*対応する脅威: 横断的。TH-10 とガバナンス。マニュアル: 第8章・第11章。*

- [ ] **18.** **粒度の細かいキルスイッチ**があり、個別のケイパビリティ — 単一のツール、MCP サーバー、外部送信、メモリ書き込み、RAG、委任、スケジュール実行 — を全体を止めずに独立して失効できるか？ `REQ-042 → 第8章`
- [ ] **19.** ランタイムセキュリティのループは**ケイパビリティを縮小する方向にしか動かず**、通常状態への復帰は自動でも暗黙でもなく、**意図的なレビューを通じてのみ**行われるか？ `REQ-046, REQ-047 → 第8章`
- [ ] **20.** 各エージェントには**文書化されたガバナンス** — ユースケース、禁止用途、責任を負うオーナー、受容したリスク、例外、適用される義務 — があるか？ `REQ-048 → 第11章`
